# Правила кодирования и стиля проекта

Этот репозиторий содержит стандарты и рекомендации по написанию и оформлению кода, принятые в нашей команде. Соблюдение этих правил помогает сделать код читаемым, поддерживаемым и единообразным.

---

## Основные принципы

### 1. Разделение статических и динамических строк

- Статические строки (без переменных) должны быть заключены в одинарные кавычки `'...'`
- Строки с динамическими данными — в обратные кавычки `` `...${переменная}...` ``

**Пример:**

```js
// Статическая строка
const message = 'Размер пакета должен быть минимум 8 байт!';

// Динамическая строка
const sizeInfo = `Текущий размер: ${packetBuffer.length}`;
```
